# Contributing to MumbAIR

First off, thank you for considering contributing to MumbAIR! It's people like you that make open-source civic tech powerful.

## How Can I Contribute?

### Reporting Bugs
* Ensure the bug was not already reported by searching on GitHub under [Issues].
* If you're unable to find an open issue addressing the problem, open a new one. Be sure to include a title and clear description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements
* Open a new issue with a clear title and description of the suggested enhancement.
* Explain why this enhancement would be useful to the citizens using MumbAIR.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Update the documentation if you change frontend behavior or UI components.
4. Ensure your code follows the standard formatting (vanilla JS/CSS best practices).
5. Issue that pull request!

## Code Style
* **HTML:** Use semantic HTML5 tags.
* **CSS:** Use CSS variables (`:root`) for colors to maintain the existing MumbAIR design system. Keep styles modular.
* **JS:** Ensure variables are scoped properly. Use clear, descriptive function names.
